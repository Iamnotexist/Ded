Это руководство по программам dead/unded.
Создатель: Артемий Морозов.
Телеграмм: @tokreal

Программа работает лишь с С++ (работает на версии ISO 14 и выше)
Требования: Windows 10, путь к g++ в переменной среды (d:\Mingw\bin).
Так же перед началом работы с программой требуется добавать в переменную среды путь до папки с программой ded
Для компиляции всех программ используйте build.bat

Руководство по настройке.
Программа настраивается через файл ded.cfg
В файле должна быть лишь одна строка.
by_co=x 

где x - колво байтов, которое хранит программа для сжатия файлов.
Чем больше x - тем потенциально эффективнее будет производится сжатие, однако программа будет съедать больше оперативной памяти.

Что делает эта программа и как с ней работать.
ded x 
сжимает файл x, где x - полный путь до файла, который нужно сжимать. (в названии файла допускаются пробелы)
файл в жатом формате будет находится в той же директории, что и сжимаемый файл и иметь расширение .ded
Если до этого существовал файл с аналогичным названием, он будет утерян.

ВАЖНО! Программа работает лишь с бинарными файлами, то есть с файлами, состоящими исключительно из 1 и 0.


unded x
разжимает файл x, где x - полный путь до файла, который нужно разжимать. (в названии файла допускаются пробелы)
разжатый файл будет находится в той же директории, где находится разжимаемый файл.
Если до этого существовал файл с аналогичным названием, он будет утерян.
Программа работает лишь с файлами, сжатами программой ded.

Стоит отметить, что программа unded защищена от "бомбы", т.е. от огромного файла, который мог получаться при разжатии.
